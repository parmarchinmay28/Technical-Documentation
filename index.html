DSA Array HashMap Stack Queue Priority Queue(Heap)

DSA

  • DSA stands for Data Structures & Algorithms

  • Data Structures are the structures in which we store data

Algoriths are how we store data

Array

  • A fixed-size, indexed data structure used to store elements of the same type.

  • Provides O(1) access time but resizing is not allowed after declaration.

int[] arr = new int[5]; // declaration int[] arr2 = {1, 2, 3}; // initialization with values

HashMap

  • Stores key-value pairs with average O(1) time for insertion, deletion, and lookup.

  • Keys must be unique, and hash functions are used internally.

HashMap map = new HashMap<>();

Stack

  • LIFO (Last-In-First-Out) data structure useful for undo operations, recursion, etc.

  • Push and pop operations are done from the top of the stack.

Stack stack = new Stack<>();

Queue

  • FIFO (First-In-First-Out) structure where elements are added at the rear and removed from the front.

  • Used in scheduling and breadth-first search (BFS).

Queue queue = new LinkedList<>();
